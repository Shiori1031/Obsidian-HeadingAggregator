# Heading Aggregator Plugin

一个用于聚合 Obsidian 笔记中指定标题下内容的插件。

## 功能特性

- ✅ **扫描所有文件**：自动遍历仓库中的所有 Markdown 文件
- ✅ **标题匹配**：查找指定标题（如 `## 项目进度`）在所有文件中的出现位置
- ✅ **多标题搜索**：支持同时搜索多个标题（如同时搜索 "introduction" 和 "引言"）
- ✅ **部分匹配**：支持匹配包含指定文本的标题（处理序号和前缀，如 `1. Introduction`）
- ✅ **内容提取**：获取该标题下的所有内容，直到遇到下一个同级或更高级别标题
- ✅ **多种展示格式**：支持列表、表格和完整 Markdown 三种展示方式
- ✅ **可点击链接**：每个结果都包含文件链接，点击即可跳转
- ✅ **灵活匹配**：可选择匹配所有级别或仅匹配指定级别的标题

## 使用方法

在任意 Markdown 文件中创建一个代码块：

```heading-agg
heading: "## summary | ## 总结 | ## 小结"
format: table
matchMode: partial
strictLevel: false
```

## 参数说明

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `heading` | 要聚合的标题（必填），可用逗号或竖线分隔多个标题 | - | `"## 项目进度"` 或 `"## introduction, ## 引言"` |
| `format` | 展示格式 | `list` | `list` / `table` / `markdown` |
| `matchMode` | 匹配模式 | `exact` | `exact`（精确）/ `partial`（部分） |
| `strictLevel` | 是否严格匹配标题级别 | `false` | `true` / `false` |
| `maxDepth` | 最大搜索深度 | `10` | `10` |

## 安装方法

1. 将插件文件夹复制到 `.obsidian/plugins/` 目录
2. 在 Obsidian 设置中启用"Heading Aggregator"插件
3. 重启 Obsidian（如需要）

## 技术细节

- 基于 Obsidian Plugin API 开发
- 使用原生 Markdown 解析
- 支持深色/浅色主题
- 响应式设计，支持移动端
- **表格优化**：
  - 紧凑布局，字体 0.9em，减少内边距
  - 智能列宽分配，内容列占主要空间
  - 美化的水平滚动条（宽度 8px，圆角）
  - 固定表头（sticky header）
  - 小屏幕自适应（<1024px 字体 0.85em）
  - 最小宽度 600px，防止过度压缩

## 注意事项

- **匹配模式**：
  - `matchMode: exact`（默认）- 精确匹配标题文本
  - `matchMode: partial` - 标题包含指定文本即可
- **标题格式**：必须包含 `#` 符号和标题级别
- **大小写敏感**：标题匹配区分大小写
- **内容提取**：提取内容时不包含标题本身
- **停止条件**：遇到同级或更高级别的标题时停止提取
- **空结果**：如果文件中没有匹配的标题，该文件不会出现在结果中
